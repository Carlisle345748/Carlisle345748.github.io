计算机储存浮点数的方式

发表于 2020-06-13

原理

浮点数在计算机中储存方式遵循IEEE 754浮点数计数标准，可以表示尾数+阶码（有效数字+指数位）：
$$
1.M…\times2^E
$$

计算机中浮点数的储存结构如下图所示，包括符号位(S)，阶码(E)和尾数(M)

  • 符号部分(s)

    0-正，1-负

  • 阶码部分(E)

    • 对于float：指数部分为8位，考虑正负和保留全为0和全为1的情况为特殊值后，指数范围为-126~127。对于有singed float，假设指数部分为10，则实际应储存为127-(-10)=137。

    • 对于double：指数部分为11位，考虑正负和保留全为0和全为1的情况为特殊值后，指数范围为-1022~1023。对于有singed float，假设指数部分为10，则实际应储存为1024-(-10)=1034。

  • 尾数部分(M)

    除全为0的情况，尾数部分的整数部分为一定为“1“，可不需储存。实际储存的是小数点后的数字。

    • 对于float：实际储存的尾数部分有23位，换算为十进制就是2^23=8388608，所以十进制精度只有6~位
    • 对于double，实际储存的尾数部分有52位，换算为十进制就是2^52=4503599627370496，所以十进制精度只有15~16位

例子

以十进制的6.36为例，十进制6.36转化为二进制后为110.010111000010100011110..，二进制小数可表示为
$$
1.10010111000010100011110..\times2^2
$$
若以float类型储存：

  • 符号部分(s)：0
  • 阶码部分(e)：127-(-2)=129，二进制表示为10000001
  • 尾数部分(M)：第一个整数”1”不储存，截去23位后的部分，尾数部分为10010111000010100011111

表达式和语句

表达式(expression)

表达式（expression）由运算符(operator)和运算对象(operand)组成。最简单的表达式是一个单独的运算对象，以此为基础可以建立复杂的表达式。

每个表达式都有一个值。要获得这个 值，必须根据运算符优先级规定的顺序来执行操作。对于有=赋值运算符的表达式，其表达式的值与赋值运算符的左侧变量的值相同。例如，a = 1 + 3表达式的值为4。对于关系运算符，其结果为真则值为1，假则值为0，例如，q>3表达式的条件为真，整个表达式的值为1。

运算对象可以是常量、变量或二者的组合。一些表达式由子表达式（subexpression）组成（子表达式即较小的表达式）。例如，c/d是上面例子中a*(b + c/d)/20的子表达式。

例子中最后一个表达式虽然很奇怪，但是是有效的。子表达式c = 3 + 8的值为11，整个表达式的值为17。

语句(statement)

语句（statement）是C程序的基本构建块。一条语句相当于一条完整的计算机指令。在C中，语句的标志是其末尾的分号。number=6只是一个表达式，而number=6;是一条语句。最简单的语句是空语句：;。

常见的语句有：声明语句(declaration statement)、赋值语句(assignment statement)、函数语句(function statement)、返回语句(return statement)等。

副作用(side effect)和序列点(sequence point)

副作用

在C语言中，术语“副作用”是对数据对象或文件的修改。例如，sum=50;的副作用是将sum设置为50，对于C语言，语句运行的主要目的是对表达式求值，表达式的副作用是对变量赋值。跟赋值运算符一样，递增和递减运算 符也有副作用，使用它们的主要目的就是使用其副作用。

序列点

序列点（sequence point）是程序执行的点，在该点上，所有的副作用都在进入下一步之前发生。在 C语言中，语句中的分号标记了一个序列点。意思是，在一个语句中，赋值运算符、递增运算符和递减运算符对运算对象做的改变必须在程序执行下一条语句之前完成。

另外，任何一个完整表达式的结束也是一个序列点。完整表达式（full expression），就是指这个表达式不是另一个更大表达式的子表达式。例如，表达式语句中的表达式和 while循环中的作为测试条件的表达式。

int类型的运算

对于int类型，运算中的小数部分会被截取，而且在连续运算中，每一个运算符的运算后，结果都会被截取小数部分后，再运行下一个运算符的运行。比如30/4*5的运算中，30/4的结果为7.5，截取小数部分为7，然后7*5等于35。

运算中的自动类型转换

通常，在语句和表达式中应使用类型相同的变量和常量。但是，如果使用不同类型的变量，C 不会像 Pascal那样停在那里卡死，而是采用一套规则进行自动类型转换。虽然这很便利，但也有一定的危险性。

  • 无论是unsigned还是signed char和short都会被自动转换成int，如有必要会被转换成unsigned int（如果short与int的大 小相同，unsigned short就比int大。这种情况下，unsigned short会被转换成unsigned int）。在K&R C中，float会被自动转换成double（目前的C不是这样）。由于都是从较小类型转换为较大类型，所以这些转换被称为升级 （promotion）;

  • 涉及两种类型的运算，两个值会被分别转换成两种类型的更高级别。类型的级别从高至低依次是long double、double、float、unsigned long long、long long、unsigned long、long、unsigned int、int。例外的情况是，当 long和int 的大小相同时，unsigned int比long的级别高。之所以short和char类 型没有列出，是因为它们已经被升级到int或unsigned int;

  • 在赋值表达式语句中，计算的最终结果会被转换成被赋值变量的类型。这个过程可能导致类型升级或降级（demotion）。所谓降级，是指把一 种类型转换成更低级别的类型。

  • 当作为函数参数传递时，如果不在函数原型（function prototype）中指定类型，char和short会被转换成int，float被转换成double。函数原型会覆盖自动升级。

  • 混合类型的表达式中，低级别的类型的数值要转化为高级别的类型，整个表达式需再统一的类型中运算。但是根据运算法优先级（precedence）和结合律（Associativity），如果优先运算的两个运算对象的类型是相同的，则在这个运算前不会先进行类型转换，而是按原有的类型运算，按原有类型储存结果，然后再转化为更高级的类型再与后续的数值运行。比如，int x = 3 / 5 * 20.0​中，3和5类型都是int，而且他们优先运算，则3/5的结果为0.6，转化为int类型阶段为0，然后0转化为double类型，然后0.0*20.0=0.0。

包含逻辑运算符的表达式的求值顺序

除了两个运算符共享一个运算对象的情况外，C 通常不保证先对复杂表达式中哪部分求值。例如，apples=(5+3)*(9+6)，可能先对表达式5 + 3求值，也可能先对表达式9 + 6求值。

但是，对于逻辑运算符是个例外，C保证逻辑表达式的求值顺序是从左往右。&&和||运算符都是序列点，所以程序在从一个运算对象执行到下一个运算对象之前，所有的副作用都会生效。而且，C 保证一旦发现某个元素让整个表达式无效，便立即停止求值。

整数(int)

储存空间不同的int类型

C语言规定，规定了short占用的存储空间不能多于int，long占用的存储空间不能少于int，但是没有规定int占用的储存空间是多少，这是因为要适配字(wrod)单位不同的计算机。现在，个人计算机上最常见的设置为：

  • short : 16位
  • int : 16位或32位
  • long ： 32位
  • long long : 64位

整数类型常数的默认类型

默认整数常数的类型是int，随着数值的增大，超出当前类型的数值范围后就会自动转化为范围更大的整数类型，转化的顺序如下:

int -> long -> unsigned long-> long long -> unsigned long long

当然，可以指定整数常数使用什数据类型，通过在常数后面加后缀（大写或小写字母l、L、UL、ull）等，可以告诉编译器该使用什么数据类型。

e.g. 5ll 10llU 9ull 12L

整数打印

浮点数精度和范围

1.范围

float和double的范围是由指数的位数来决定的。 float的指数位有8位，而double的指数位有11位，分布如下：

于是，float的指数范围为-127至+128 (2^7=128)，double的指数范围为-1023至1024 (2^10=1024)。但是各指数全为1或全为0的情况是用于表示某些特殊数值，因此实际中float的指数范围为-126至+127，double的指数范围为-1022至1023。

  • float

    • 最大正数

      $$
      (2 - 2^-{23})\times2\times^{127}=3.402823466\times10^{38}
      $$

    • 最小正数

      $$
      (1 + 2^-{23})\times2\times^{-126}=1.175494351\times10^{-38}
      $$

  • double

    • 最大正数

      $$
      (2 - 2^-{52})\times2\times^{1023}=1.7976931348623157\times10^{308}
      $$

    • 最小正数

      $$
      (1 + 2^-{23})\times2\times^{-126}=2.225073858507202\times10^{-308}
      $$

2.精度

float和double的精度是由尾数的位数来决定的。浮点数在内存中是按科学计数法来存储的，其整数部分始终是一个隐含着的“1”，由于它是不变的，故不能对精度造成影响。

  • float：2^23 = 8388608，一共七位，这意味着最多能有7位有效数字，但绝对能保证的为6位，也即float的精度为6~7位有效数字；
  • double：2^52 = 4503599627370496，一共16位，同理，double的精度为15~16位。（能绝对保证15位没有问题）

字符串

储存结构

C语言没有专门用于储存字符串的变量类型，字符串都被储存在char类型的数组中。数组由连续的存储单元组成，字符串中的字符被储存在相邻的存储单元中，每个单元储存一个字符。字符串数组的结尾\0为空字符（null character），\0不是数字零而，是非打印字符，其对应的ASCII 码为0。C中的字符串一定以空字符结束，这意味着数组的容量必须至少比待存储字符串中的字符数多1。

字符串的strlen()和sizeof()结果

strlen()函数包含在string.h这个头文件(header file)中，需要在程序开头声明。sizeof()运算符报告，name数组有40个存储单元。但是，只有前8个单元用来储存Carlisle，所以strlen()得出的结果是8。name数组的第12个单元储存空字符，strlen()并未将其计入。C99 和 C11 标准专门为 sizeof()运算符的返回类型添加 了%zd转换说明，这对于strlen()同样适用。

字符串与字符数组

字符串以数组的方式储存，数组中包含字符和末尾的空字符(\0)；字符数组也以数组的方式储存，但是其末尾没有空字符(\0)。末尾有空字符(\0)的字符数组为字符串。

sizeof与sizeof()

当sizeof运算对象是类型(char/int/float等)时，圆括号必不可少。但是对于特定量(变量/常量)，圆括号可有可无。尽管如此，还是建议所有情况下都使用圆括号，如sizeof(6.28)。

  • 类型：应写成sizeof(char)或sizeof(float)；
  • 特定量：可写成sizeof name或sizeof 6.28，但是建议加上圆括号。

目标代码文件(Object Code File)、可执行文件(Executable File)和库(Libraries)

典型的C实现通过编译(compile)和链接(link)两个步骤来完成这一过程。编译器把源代码转换成中间代码，链接器(linker)把中间代码和其他代码合并，生成可执行文件。

目标代码文件是C语言代码直接翻译成的机械语言(machine language)。目标代码文件无法直接运行，为运行程序，目标代码文件还需要：

  • 启动代码(startup code)。启动代码充当着程序和操作系统之间的接口，不同的操作系统对应的启动代码不同。
  • 库函数(libraries routines)。几乎所有的C程序都要使用C标准库中的函
    数，如printf()函数。printf()函数真正的代码储存在另一个被称为库的文件中。库文件中有许多函数的目标代码。

链接器(linker)把目标代码文件、对应操作系统的启动代码和所使用的的库函数的代码合并成一个文件(可执行文件)，程序才可运行。

GUN编译器和LLVM项目

GNU项目始于1987年，是一个开发大量免费UNIX软件的集合（GNU的意思是“GNU’s Not UNIX”，即GNU不是UNIX）。GNU编译器集合（也被称为GCC，其中包含GCC C编译器）是该项目的产品之一。

LLVM项目成为cc(Unix C编译器，已不再使用)的另一个替代品。该项目是与编译器相关的开源软件集合，始于伊利诺伊大学的2000份研究项目。它的 Clang编译器处理 C代码，可以通过 clang调用。有多种版本供不同的平台使用，包括Linux。2012年，Clang成为FreeBSD的默认C编译器。Clang也对最新的C标准支持得很
好。

PC上编译C语言

Windows不自带C编译器，可以下载Cygwin或MinGW来使用GCC。Cygwin有自己的窗口，MinGW使用Windows自带的命令行窗口。PC上的C编译器的中间文件名为(.obj)，不像Unix系统，Windows不会自动删除中间文件而会保留下来，有些编译器会生成(.asm)的汇编语言文件或其独特的文件格式。可执行文件的扩展名为(.exe)。

微软等公司都在Windows上提供了集成式开发环境(Integrated Development ENvironment, IDE)，这些IDE都内置了用于编写C程序的编辑器。这类集成开发环境都提供了各种菜单（如，命名、保存源代码文件、编译程序、运行程序等），用户不用离开IDE就能顺利编写、编译和运行程序。如果编译器发现错误，会返回编辑器中，标出有错误的行号，并简单描述情况。

Mac OS X 上编译C语言

Mac是基于Unix系统开发的类Unix系统(Unix-like)，其目标代码文件和可执行文件的拓展名与Unix和Linux系统相同。Mac中可以使用Xcode或CLion等IDE编写和调试C语言,。Xcode使用Clang或GCC,现在默认使用Clang。可以设置选择使用哪一个编译器和哪一套C标准。Mac的terminal与Unix和Linux命令行几乎一致，但Mac系统不自带命令行编译器。但如果下载了Xcode，还可以下载可选的命令行工具(command-line tool)，这样就可以使用Clang和GCC命令在terminal中编译。

标识符名(变量名、函数名等)的长度限定

C99和C11允许使用更长的标识符名，但是编译器只识别前63个字符。对于外部标识符，只允许使用31个字符。〔以前C90只允许6个字符，这是一个很大的进步。旧式编译器通常最多只允许使用8个字符。〕实际上，你可以使用更长的字符，但是编译器会忽略超出的字符。也就是说，如果有两个标识符名都有63个字符，只有一个字符不同，那么编译器会识别这是两个不同的名称。如果两个标识符都是64个字符，只有最后一个字符不同，那么编译器可能将其视为同一个名称，也可能不会。标准并未定义在这种情况下会发生什么。

位(bit)、字节(byte)、字(word)

1字节(byte)=8位(bit)。字（word）是设计计算机时给定的自然存储单位。对于8位的微型计算机（如，最初的苹果电脑）， 1个字长只有8位。从那以后，个人计算机字长增至16位、32位，直到目前的64位。计算机的字长越大，其数据转移越快，允许的内存访问也更多。

实际参数(argument)和形式参数(parameter)

声明pound函数的时候，创建了形式参数（formal argument或formal parameter）n, 类型为int，像pound(10)这样调用函数会把10这个值赋给n，调用pound(times)时就把times的值5赋给n。我们把函数调用传递的值称为实际参数（actual argument或actual parameter）。

在英文中，argument和parameter经常可以互换使用，但是C99标准规定了：对于actual argument或actual parameter使用术语argument（译为实参）；对于formal argument或formal parameter使用术parameter（译为形参）。为遵循这一规定，我们可以说形参是变量，实参是函数调用提供的值，实参被赋给相应的形参。
