目标代码文件(Object Code File)、可执行文件(Executable File)和库(Libraries)

典型的C实现通过编译(compile)和链接(link)两个步骤来完成这一过程。编译器把源代码转换成中间代码，链接器(linker)把中间代码和其他代码合并，生成可执行文件。

目标代码文件是C语言代码直接翻译成的机械语言(machine language)。目标代码文件无法直接运行，为运行程序，目标代码文件还需要：

  • 启动代码(startup code)。启动代码充当着程序和操作系统之间的接口，不同的操作系统对应的启动代码不同。
  • 库函数(libraries routines)。几乎所有的C程序都要使用C标准库中的函
    数，如printf()函数。printf()函数真正的代码储存在另一个被称为库的文件中。库文件中有许多函数的目标代码。

链接器(linker)把目标代码文件、对应操作系统的启动代码和所使用的的库函数的代码合并成一个文件(可执行文件)，程序才可运行。

GUN编译器和LLVM项目

GNU项目始于1987年，是一个开发大量免费UNIX软件的集合（GNU的意思是“GNU’s Not UNIX”，即GNU不是UNIX）。GNU编译器集合（也被称为GCC，其中包含GCC C编译器）是该项目的产品之一。

LLVM项目成为cc(Unix C编译器，已不再使用)的另一个替代品。该项目是与编译器相关的开源软件集合，始于伊利诺伊大学的2000份研究项目。它的 Clang编译器处理 C代码，可以通过 clang调用。有多种版本供不同的平台使用，包括Linux。2012年，Clang成为FreeBSD的默认C编译器。Clang也对最新的C标准支持得很
好。

PC上编译C语言

Windows不自带C编译器，可以下载Cygwin或MinGW来使用GCC。Cygwin有自己的窗口，MinGW使用Windows自带的命令行窗口。PC上的C编译器的中间文件名为(.obj)，不像Unix系统，Windows不会自动删除中间文件而会保留下来，有些编译器会生成(.asm)的汇编语言文件或其独特的文件格式。可执行文件的扩展名为(.exe)。

微软等公司都在Windows上提供了集成式开发环境(Integrated Development ENvironment, IDE)，这些IDE都内置了用于编写C程序的编辑器。这类集成开发环境都提供了各种菜单（如，命名、保存源代码文件、编译程序、运行程序等），用户不用离开IDE就能顺利编写、编译和运行程序。如果编译器发现错误，会返回编辑器中，标出有错误的行号，并简单描述情况。

Mac OS X 上编译C语言

Mac是基于Unix系统开发的类Unix系统(Unix-like)，其目标代码文件和可执行文件的拓展名与Unix和Linux系统相同。Mac中可以使用Xcode或CLion等IDE编写和调试C语言,。Xcode使用Clang或GCC,现在默认使用Clang。可以设置选择使用哪一个编译器和哪一套C标准。Mac的terminal与Unix和Linux命令行几乎一致，但Mac系统不自带命令行编译器。但如果下载了Xcode，还可以下载可选的命令行工具(command-line tool)，这样就可以使用Clang和GCC命令在terminal中编译。

标识符名(变量名、函数名等)的长度限定

C99和C11允许使用更长的标识符名，但是编译器只识别前63个字符。对于外部标识符，只允许使用31个字符。〔以前C90只允许6个字符，这是一个很大的进步。旧式编译器通常最多只允许使用8个字符。〕实际上，你可以使用更长的字符，但是编译器会忽略超出的字符。也就是说，如果有两个标识符名都有63个字符，只有一个字符不同，那么编译器会识别这是两个不同的名称。如果两个标识符都是64个字符，只有最后一个字符不同，那么编译器可能将其视为同一个名称，也可能不会。标准并未定义在这种情况下会发生什么。

位(bit)、字节(byte)、字(word)

1字节(byte)=8位(bit)。字（word）是设计计算机时给定的自然存储单位。对于8位的微型计算机（如，最初的苹果电脑）， 1个字长只有8位。从那以后，个人计算机字长增至16位、32位，直到目前的64位。计算机的字长越大，其数据转移越快，允许的内存访问也更多。

实际参数(argument)和形式参数(parameter)

声明pound函数的时候，创建了形式参数（formal argument或formal parameter）n, 类型为int，像pound(10)这样调用函数会把10这个值赋给n，调用pound(times)时就把times的值5赋给n。我们把函数调用传递的值称为实际参数（actual argument或actual parameter）。

在英文中，argument和parameter经常可以互换使用，但是C99标准规定了：对于actual argument或actual parameter使用术语argument（译为实参）；对于formal argument或formal parameter使用术parameter（译为形参）。为遵循这一规定，我们可以说形参是变量，实参是函数调用提供的值，实参被赋给相应的形参。
